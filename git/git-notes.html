Git and Github

Git is a version control system which allows you to manage source code history. It can be downloaded, so it's a local tool that you install to your machine. It's possible to push your code to Github for others to see. There's no interface - it is ran by terminal.

  • Repository - location where your code history is stored
  • Branch - "Folder" within the repository containing commits
  • Commits - Different stages of your code inside a branch

    • Command

    $git init

    Initializes a local Git in a repository

    $git status

    Displays the state of the working directory and the staging area

    $git add .

    Adds a change in the working directory to the staing area

    $git commit -m "..."

    Saves changes to the local repository

    $git branch

    Creates a master branch

    $git log

    Detailed history of repository

    $git checkout (+commit id)

    Switches the currently active branch. Best to use if you want to check previous code

    $git master

    Switches to last commit or master branch

    $git reset --hard ...

    Moves current Head to commit specified, add commit id to the end

    $git checkout -- .

    Reverts saved changes

    $git checkout -b branch-name

    Switches to new branch (not master) that contains everything from master branch

    $git merge (+new branch name)

    Merges two branches. Enter commit while in master branch.

    $git branch -D branch-name

    Deletes specified branch

    Github is a hosting and collaboration provider which has the purpose to host your git repositories. It's a web based solution, so it's a remote tool. An account is needed in order to use it. Create a repository in Github beforehand.

    Command

    $git remote add origin ...

    Add a link to your github repository. Creates a connection between local and remote repository

    $git push -u origin master

    Pushes branches to remote repository (to Github)

    $git clone

    Copies an existing Git repository

    $git push

    Same as mentioned before, except

    $git fetch

    Allows to get the code from Github. It just syncs code, needs merging after!

    $git pull

    Same as fetch, except it merges master/origin for you and you'll get the code to local
